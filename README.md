Бізнес-логіка реалізована автоматично при створенні дзвінка через **Observer (CallObserver)**:
- Перший дзвінок переводить лід зі статусу `new` у `in_progress`
- Якщо у ліда ще немає менеджера — він автоматично закріплюється за менеджером, який зробив дзвінок
- Якщо результат дзвінка `success` — статус ліда стає `won`
- Якщо останні 3 дзвінки підряд мають результат `no_answer` — статус ліда стає `lost`
- Статус ліда не може змінюватися напряму через API
